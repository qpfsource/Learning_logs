一、用户相关配置文件
		1.  etc/passwd------->用户基本信息文件，用户的基本信息存储在这个文件中
		    字段信息
					字段1：用户名称
					字段2：密码标志
					字段3：UID（用户ID）
					查看配置文件的帮助信息         使用命令   man 5 配置文件名称
							  0：                  超级用户   root     将用户的UID改为0后，用户就会变成超级用户
							  1-499:               系统用户（伪用户）  不能进行操作，更不能删除
							  500-65535：		   普通用户	
					字段4：GID------->用户的初始组ID（初始组和附加组）,每个用户初始组必须有且必须唯一
						   附加组:用户可以加入多个其他用户组，并拥有这些附加组的权限，附加组可以有多个
					字段5：用户的解释说明信息
					字段6：用户的家目录
					字段7：登录之后的shell       修改用户的shell为sbin/nologin后，该用户将无法登录
		2.etc/shadow-------->用户影子文件，保存的是用户的密码信息
		    字段信息
					字段1: 用户名称
					字段2：经过加密处理后的用户密码 (使用的是SHA512散列加密算法，如果密码为"!!"或"*",则代表没有密码，不能登录)
					字段3：密码的最后一次修改日期（使用1970年1月1日为标准时间，每过一天时间戳加1）
					字段4：两次密码修改的间隔时间,设置间隔时间后只有在指定的时间后才可以进行密码修改
					字段5：密码的有效时间，设置密码有效期之后，到期之后用户无法登录
					字段6：密码修改到期前的警告天数（和字段5相比）
					字段7：密码过期后的宽限天数（和字段5相比，0代表过期后立即失效，-1代表密码永久不会失效）
					字段8：账号失效时间
					字段9：保留
					时间戳和日期的相互换算：
						时间戳换算为日期：date -d "1970-01-01 16066 days"
						日期换算为时间戳：echo $(($(date --date="2014/01/06" +%s)/86400+1))
		3.etc/group--------->组信息文件，保存的是用户的组信息
			字段信息：
					字段1：组名
					字段2：组密码标志
					字段3：GID,组ID
					字段4：组中的附加用户
		4.etc/gshadow-------->组密码文件（不推荐使用）
			字段信息：
					字段1：组名
					字段2：组密码
					字段3：组管理员用户名
					字段4：组中的附加用户
二、用户管理的相关文件
		1.用户的家目录：/home/用户名/,所有者和所属组都是此用户，权限是700    超级用户：/root/，所有者和所属组都是root用户，权限是500
		2.用户的邮箱：/var/spool/mail/用户名/
		3.用户模板目录：/etc/skel/(添加用户后显示的默认信息，比如添加一个警告信息！) cd /etc/skel   vim warning.txt("请遵守服务器的操作规范！") -->添加用户登录后查看家目录
		添加一个用户，就会自动修改添加以上七个文件的内容！！！
		
三、用户管理命令
			1.用户添加命令   useradd  
				选项：-u UID                    手动指定用户的UID号
				      -d 家目录(dictory)：      手动指定用户的家目录
					  -c 用户说明(comment)：    手动指定用户的说明信息
					  -g 组名：                 手动指定用户的初始组
					  -G 组名：                 手动指定用户的附加组
					  -s shell:                 手动指定用户的登录shell.默认是/bin/bash
						  
						添加用户user1    useradd user1    
						查看相关信息：
											  1.grep user1 /etc/passwd
											  2.grep user1 /etc/shadow
											  3.grep user1 /etc/group
											  4.grep user1 /etc/gshadow
											  5.ll-d /home/user1/
											  6.ll /var/spool/mail/user1
			2.用户默认值的配置文件
					1./etc/default/useradd
							GROUP                    用户默认组
							HOME                     用户默认家目录
							INCATIVE=-1              密码过期天数
							EXPIRE                   密码失效时间
							SHELL=/bin/bash          默认shell
							SKEL=/etc/skel           模板目录
							CAEATE_MAIL_SPOOL=yes    是否建立邮箱
							
							
					2./etc/login.defs
					        PASS_MAX_DAYS       9999                          密码有效期
							PASS_MIN_DAYS        0                            密码修改间隔
							PASS_MIN_LEN         5                            密码最小5位
							PASS_WARN_AGE        7                            密码到期警告
							UID_MIN              500                          最小和最大的UID范围
							GID_MAX             60000                   
							ENCRYPT_METHOD      SHA512                            加密模式  
			3.用户管理命令  passwd                      passwd -选项     用户名     查看当前登录用户   whoami
			        选项：
							-S                         查询用户的密码状态，仅root可用
							-l                         暂时锁定用户。仅root 可用
							-u                         解锁用户，仅root 可用
							--stdin                    可以通过管道符输出的数据作为用户的密码   echo "123" |passwd --stdin user1
			
			4.修改用户信息    usermod  选项  用户名（修改已存在的用户）
									选项：-u UID                    修改用户的UID号
										  -d 家目录(dictory)：      修改用户的家目录
										  -c 用户说明(comment)：    修改用户的说明信息
										  -g 组名：                 修改用户的初始组
										  -G 组名：                 修改用户的附加组
										  -L                         临时锁定用户
										  -U                         解锁用户
										  
			5.修改用户密码状态   chage   选项    用户名         chage -l user1     chage -d 0 user1(设置用户user1登录后就必须修改密码)
						选项 ：   所有的选项可以不用记
									-l                                  列出用户的详细密码状态
									-d 日期                             修改密码的最后一次修改日期
									-m 天数                             两次修改密码的时间间隔
									-M 天数                             密码有效期
									-W 天数                             密码过期前的警告天数
									-I 天数                             密码过期后的宽限天数
									-E 日期                             账号的失效时间
									
			6.删除用户及用户切换
						删除用户     userdel    用户名   选项 -r   代表删除用户时同时删除用户的家目录
						查看用户  ID   id user1   查看user1的ID信息
						切换用户  su - 用户名  
									选项：
												 -  选项只使用"-"则代表连同用户的环境变量一起切换
									             - c 命令：仅执行一次，而不切换用户身份    su -root -c "useradd user1"(不切换到root身份的前提下调用一次root添加用户的命令)
						env     查看用户的环境变量
			7.用户组管理命令
						1.groupadd  选项    组名    （选项-g GID 指定组ID）
						2.groupmod  选项    组名   （-g GID 修改组ID  -n 新组名，修改组名）   groupmod -n test develop  修改develop组名为test
						3.groupdel  组名    
						4.将已存在的用户加入到对应的组中   gpasswd  选项  组名（-a用户名，将用户加入到对应的组  -d 用户名  将用户从指定的组中删除）
						注：一个文件只能有一个所属组
