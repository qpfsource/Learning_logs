一、文件特殊权限
		1.SetUID
							1.可执行的二进制程序，可执行程序。   passwd命令
							2.命令执行者必须对命令有执行权限。
							3.命令执行者在执行过程中获得改程序的属主身份。（在执行命令的过程中变成命令的所有者）
							4.SetUID 权限只在该程序执行过程中有效。
							设置SUID   方法：    chmod   4755   文件名     GID     2755   文件名     StickyBIT   1755  文件名  4:代表SUID 2：代表GID 1:SBIT
							chmod u+s 文件名
							注意：这种权限特别可怕，不建议使用！！！
					
		2.SetGID
							1.可执行的二进制程序，可执行程序。   passwd命令
							2.命令执行者必须对命令有执行权限。
							3.命令执行者在执行过程中获得改程序的属组身份。（在执行命令的过程中变成命令的所属组）
							4.SetGID 权限只在该程序执行过程中有效。
						例：locate     /usr/bin/locate
						注意：SetGID可以针对目录
						针对目录的SGID
								1.普通用户必须对该目录拥有r和x权限，才能进入此目录
								2.普通用户在此目录中的有效组就会变成此目录的属组
								3.若普通用户对该目录有w权限时，新建的文件也会属于当前目录的属组
						
		3.Sticky BIT
						1.操作的对象是目录
						2.普通用户对该目录有w和x权限，即普通用户可以在此目录中拥有写入的去权限
						3.如果没有黏着为，因为普通农用户拥有w权限，所以可以在此目录下删除所有文件，一旦赋予了黏着位，除了root用户可以删除中所有的文件，
						普通用户就算有w权限，也只能删除自己创建的文件，不能删除其他用户创建的文件。
						 例：ll -d /tmp/
						 设置：chmod 1755  目录名
						       chmod o+t   目录名
						 取消：chmod 777   目录名
						       chmod  o-t  目录名
二、文件系统属性chattr 权限
		chattr [+-=][选项]  文件或目录名
		+:  增加权限
		-： 删除权限
		=： 等于某权限
				选项：
							i:如果对文件设置i属性，那么不允许对文件进行修改、改名、删除、也不能进行添加和修改数据；
							如果对目录设置i属性，则只能修改目录下的文件的数据，但不允许建立和删除文件
							a:如果对文件设置a属性，那么只能在文件中增加数据，但是不能删除和修改数据
							如果对目录设置a属性，那么只允许在目录中建立和修改文件，但是不允许删除
							
			lsattr [选项]文件名      对root 用户也可以限制
			选项：
			 chattr +i  文件名或目录
				a  显示所有文件和目录
				d   如果目标是目录，仅列出目录本身的属性，而不是子文件的

三、sudo权限
			root把本来只能超级用户执行的命令赋予普通用户执行
			sudo的操作对象是系统命令
			
			visudo    实际修改的是etc/sudoers文件
			root   ALL=（ALL）   ALL
			# 用户名  被管理的主机地址=（可使用身份）  授权命令（绝对路径）
			user1   ALL = /sbin/shutdown -r now      允许用户执行重启的操作命令
			
			
			# %wheel ALL=(ALL)  ALL
			# %组名  被管理的主机地址=（可使用身份）   授权命令（绝对路径）
			用户执行
			sudo -l   查看管理员赋予的命令
			sudo /sbin/shutdown -r now
